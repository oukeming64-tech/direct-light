# v0.6b 附件视觉和导演视角简介规格

状态：已落地并通过用户视觉验收。Codex 产品/视觉规格；v0.6a 控光附件 MVP 已通过用户真机验收，v0.6b 只补附件视觉和导演视角简介，不改 v0.6a 光学数值。创建日期：2026-06-20。

## 1. 目标

v0.6a 已经让附件影响有效亮度、光束角、柔硬和彩色溢光。v0.6b 不再改这些光学数值，而是补上两件沟通体验：

1. 灯上装了什么附件，在 3D 画面里应该能被看见。
2. 导演视角选中灯时，应该能快速读到“这盏灯是什么器械、装了什么、主要效果是什么”。

一句话目标：让导演不用打开右侧面板，也能从画面和一句简介里看懂当前灯光设置。

## 2. 范围

只做 v0.6a 四个灯上附件的可见表达：

- 中号柔光箱。
- 蜂巢。
- 标准反光罩。
- 柔光布。

只做选中灯的导演视角简介，不做完整教程、不做多语言、不做更多光源、不做棚内独立控光器材。

## 3. 附件视觉规则

### 3.1 中号柔光箱

视觉意图：一眼看出“这盏灯前面装了一个更大的软光面”。

建议表达：

- 在灯头前方显示一个比普通柔光灯体更大的矩形或近矩形柔光面。
- 面颜色偏白，略有发光感，但不要亮到遮挡人物。
- 尺寸应明显大于柔光布，和 `effectiveSoftness` / `effectiveBeamAngle` 有轻微关联即可。
- 不能改变灯光拖拽命中逻辑；点击可见灯体仍应能选中 / 拖动对应灯。

### 3.2 蜂巢

视觉意图：一眼看出“灯头前方有控光格栅，光更收”。

建议表达：

- 在灯头前方显示一个深色短罩或黑色网格面。
- 形体要小，贴近灯头，不要像独立棚内物体。
- 可以用暗色圆环 / 方环 + 少量细线暗示格栅，不需要真实蜂窝几何。

### 3.3 标准反光罩

视觉意图：一眼看出“裸灯前方有金属反光罩，光更集中、更硬、更亮”。

建议表达：

- 在灯头前方显示浅金属色浅锥形 / 碗形罩体。
- 体积比蜂巢略大，比柔光箱明显小。
- 不要做成独立可拖动物；它仍然只是灯的一部分。

### 3.4 柔光布

视觉意图：和中号柔光箱清楚区分；它是“薄薄一层扩散片”，不是大软箱。

建议表达：

- 在灯头前方显示一片半透明薄平面。
- 尺寸只比灯头略大，明显小于中号柔光箱。
- 透明度要能看出“布/片”的感觉，不要变成第二个白色软箱。

## 4. 导演视角简介

只在“镜头视角”里出现，且只在当前选中对象是灯时出现。

位置建议：

- 画面左下或右下角。
- 一行或两行短文本，不做大卡片，不遮挡人物主体。
- 视觉上像轻量状态条，而不是说明书。

文案规则：

- 有灯具 + 有附件：
  `Key Light · Nanlux Evoke 600C · 中号柔光箱 · 柔光主光`
- 有灯具 + 无附件：
  `Key Light · Nanlux Evoke 600C · 无附件`
- 无灯具 + 有附件：
  `Key Light · 自定义灯具 · 柔光布 · 扩散片`
- 无灯具 + 无附件：
  `Key Light · 自定义灯具 · 无附件`

最后一段“效果短语”来自附件：

| 附件 | 效果短语 |
| --- | --- |
| 中号柔光箱 | 柔光主光 |
| 蜂巢 | 收束控光 |
| 标准反光罩 | 集中硬光 |
| 柔光布 | 扩散片 |
| 无附件 | 可手动调光 |

## 5. 建议写入范围

Claude Code 主实现建议：

- `src/scene/LightVisual.tsx`：承载灯体和附件视觉。若文件开始变胖，可新增 `src/scene/LightModifierVisual.tsx` 并由 `LightVisual` 组合。
- `src/scene/LightRig.tsx`：把 modifier preset / effective values 传给可见灯体；不要改变现有灯光 effective params 公式。
- `src/app/Stage.tsx`：在镜头视角叠加导演简介。若组件变长，可新增 `src/app/DirectorLightBrief.tsx`。
- `src/domain/lightingSummary.ts` 或新增 `src/domain/lightBrief.ts`：生成导演简介文案，保持纯函数。
- `src/ui/TopBar.tsx`：若尚未同步，顶栏版本显示到 `v0.6b`。

允许读取：

- `src/data/lightModifiers.ts`
- `src/data/fixturePresets.ts`
- `src/domain/lightModifiers.ts`
- `src/domain/lightingSummary.ts`
- `src/types.ts`

不应该触碰：

- `src/data/lightModifiers.ts` 的 v0.6a 数值，除非用户明确要求重调。
- `src/domain/lightModifiers.ts` 的有效光质公式。
- `MAX_LIGHTS` 或更多光源管理。
- v0.6c/d 的旗板、黑旗、反光板、独立柔光布框。
- 后续开源文档、桌面封包、多语言（均不属于 v0.6b；开源和桌面封包后续已完成，多语言后置）。

## 7. 验收清单

基础视觉：

1. 选择中号柔光箱时，灯前方能看到明显更大的柔光面。
2. 选择蜂巢时，灯头前方能看到深色控光格栅 / 小罩体。
3. 选择标准反光罩时，灯头前方能看到金属反光罩，且不像柔光箱。
4. 选择柔光布时，能看到薄的半透明扩散片，且和中号柔光箱明显不同。
5. 清空附件后，灯体回到无附件视觉。

交互：

6. 点击 / 拖动可见灯体仍然能选中和移动灯。
7. 自由、俯视、镜头视角都能看懂附件大致形体；侧视不要求完美，但不能遮挡成异常黑块。
8. A/B 对比视图不黑屏，不因为附件视觉导致画布丢失。

导演简介：

9. 镜头视角选中灯时出现一条简短简介。
10. 选择人物、道具、摄影机或白棚时，灯光简介不显示。
11. 切换灯具预设、附件、灯名后，简介同步更新。
12. 文案短，不遮挡人物主体。

回归：

13. v0.6a 四个附件的画面差异不回退。
14. v0.5.1 可见灯体和拖拽不回退。
15. `npm run lint` 和 `npm run build` 通过。
