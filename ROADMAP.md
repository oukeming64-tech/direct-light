# Direct Light 后续需求路线图

## 1. 文档目的

这份文档把 v0.2 之后的需求拆成可交付版本，继续沿用原来的协作分工：

- Codex 负责产品/视觉/渲染规格、默认数值、验收标准和判图。
- Claude Code 负责工程实现、代码结构、状态管理、交互落地和本地验证。
- 共同负责截图验收和体验取舍。

A/B 对比已经在 v0.1.2 完成，后续需求默认都要能被 A/B 对比验证。

## 2. 后续优先级

优先顺序：

1. 人物数量与站位。
2. 白棚结构、道具、人台和现场沟通交互。
3. 人物基础姿态。
4. 灯具器械预设库。
5. 控光附件。
6. ~~v0.7 可开源第一版 / 桌面封包。~~ ✅ 已完成到 v0.7.1。
7. v0.8 多语言。
8. v0.9 更多光源 / 多灯管理。
9. 更真实的品牌参数和光度数据。

原因：

- 导演首先需要看人和影子在哪里。
- 然后需要把桌子、椅子、人台、台座、背景板这类拍摄现场会出现的结构放进白棚，方便拍摄前沟通调度。
- 再往后才需要看手、头、身体姿态是否挡光或改变阴影。
- 灯具品牌型号可以先做成语义预设，真实光度数据后补。
- 柔光箱、蜂巢、黑旗、反光板、柔光布框对画面影响很大，但需要建立在灯和人物都稳定之后。

## 3. v0.2 多人物与站位

> 状态：工程已落地（Claude Code v0.2.0，2026-06-18）；视觉规则已定稿（Codex v0.2.1，2026-06-18）；A/B 切换黑屏、侧视被侧墙遮挡、重复拖拽控制器等稳定性问题已修到 v0.2.3。加/复制/删人物、逐人选择/编辑/拖拽、多人投影、A/B 含多人均通过本地验收。默认多人站位与命名/颜色规则见 `RENDERING_SPEC.md §6`。

### 目标

让导演能快速模拟多人站位、遮挡、投影和灯光关系。

### MVP 范围

- 添加人物。
- 删除人物。
- 复制人物。
- 选择人物。
- 重命名人物。
- 调整人物位置。
- 调整人物朝向。
- 调整人物身高。
- 调整人物肤色和服装颜色。
- 最多先支持 5 个人，默认建议 3 个人以内。
- 多人物均能投射阴影。
- 多人物之间可以互相遮挡。
- 俯视图显示每个人的位置和朝向。
- A/B 对比中包含多人物状态。

### 暂时不做

- 群演级大数量人物。
- 路径动画。
- 碰撞约束和自动避让。
- 精确人物身材系统。

## 4. v0.3 白棚结构、道具和人台

> 状态：v0.3.4 已落地（2026-06-18）。Claude Code 已完成结构/道具工程接入；Codex 补齐“直播圆形小舞台”、人物离地高度/承载面放置，以及灯光 `manual/person/peopleCenter` 目标模式。下一步应进入 v0.4 人物基础姿态，或先让 Claude 做一轮 v0.3 交互打磨。

### 目标

让 Direct Light 不只模拟“人在空白棚里被灯照”，也能模拟拍摄现场常见的基础结构。这样导演、摄影、美术、制片在拍摄前可以沟通：

- 人和桌椅的相对位置。
- 灯打到桌面、椅背、人台、背景板后的阴影。
- 产品台、台座、椅子是否遮挡人物。
- 道具和结构会不会吃光、反光、投影到白棚。
- 镜头里实际看到的空间关系。

### 产品原则

这不是完整美术置景软件。第一版只做“拍摄沟通够用”的结构和道具：

- 尺寸清楚。
- 位置清楚。
- 材质大类清楚。
- 能投影、能挡光。
- 在俯视图里能看懂。

### v0.3c 服装人台与人物站台

这里要分清两个概念：

- 服装人台：mannequin，用于看服装、胸口受光和轮廓光。
- 人物站台：可承重平台，例如直播用圆形小舞台，用于把真人放高。

第一版服装人台建议：

- 半身人台。
- 全身人台。
- 服装展示人台。
- 可调高度。
- 可调朝向。
- 可调颜色：白、灰、黑、肤色、布面。
- 可投影和接收阴影。

第一版人物站台建议：

- 低矮长方形平台。
- 直播圆形小舞台，默认 `1.2m × 1.2m × 0.3m`。
- 人物可以放到台面上，脚底高度等于台面高度。

暂时不做：

- 人台复杂曲面精度。
- 可换衣服。
- 布料模拟。
- 真实品牌人台型号。

### v0.3d 现场沟通交互补齐

v0.3 不只增加物体，还要把“拍摄前沟通”需要的直接操作补齐。

人物交互：

- 人物继续支持直接拖动，拖动范围限制在白棚地面内。
- 人物支持 `position.y` 离地高度，第一版面板范围 `0..1.5m`。
- 人物可以一键放到桌面、椅面、凳面、沙发坐面、台座、箱体或直播圆形小舞台上。
- 在自由、俯视、侧视里都能选中人物；俯视是最主要的站位编辑视图。
- 右侧面板继续保留精确数值输入，用于和拖动互补。
- A/B 中改 A 的人物位置时，B 保持冻结。

灯光目标模式：

- 主人物：沿用当前 Actor A / `people[0]` 作为目标。
- 锁定人物：自动对准指定 Actor，人物移动或升高后目标随之更新。
- 多人中心：自动对准当前所有人物的目标点中心，适合群像或访谈。
- 对准一次：把当前灯目标写成某个人的位置，然后保持手动。
- 指定目标点：后续可在俯视图放一个 target marker，灯光对准该点。

白棚结构交互：

- 桌椅、台座、人台和背景板都应能拖动、旋转、复制和删除。
- 俯视图中结构对象需要显示外轮廓和朝向。
- 对象移动时实时更新遮挡、受光和投影。

## 5. v0.4 人物基础姿态

> 状态：v0.6e 已通过用户视觉验收（2026-06-21）。v0.4a（v0.4.2）层级骨架/9 预设/10 滑杆 + 迁移；v0.4.3 ParamPanel/StudioScene 拆分；v0.4.4 坐姿/承载面坐站分流/前臂内旋；v0.4.5 attach-to-support 实时绑定；v0.4.6 A/B 对比产品引导；v0.4.7 摄影机控制已通过用户真机验收；v0.4.8 按 `STORE_ACTION_SPLIT_SPEC.md` 拆 `store.ts`（685→38 行，八组 action 工厂 + storeTypes/storeHelpers，无行为变更，tsc/lint/build 全绿）。v0.5.0 灯具器械预设库已完成；v0.5.1 渲染可信度已完成；v0.6a 控光附件 MVP、v0.6b 附件视觉/导演简介、v0.6c 棚内独立控光器材、v0.6d 近似光学和 v0.6e 收口均已通过。v0.7 开源与桌面封包已完成到 v0.7.1；当前后续主线是 v0.8 多语言和 v0.9 更多光源 / 多灯管理。

### 目标

让人物不只是“站桩”，可以用基础姿态测试遮挡、轮廓光和身体阴影。

### 产品原则

Direct Light 不是角色动画软件。姿态功能服务于灯光判断：

- 手会不会挡脸。
- 头转向后脸部受光怎么变。
- 侧身后轮廓光是否明显。
- 身体前倾或转身后，影子形状是否变化。

### v0.4a 姿态预设

状态：已完成初版。

优先做预设，不先做复杂骨骼编辑。

建议预设：

- 自然站立。
- 侧身站立。
- 头转向主光。
- 低头。
- 抬一只手。
- 双手自然下垂。
- 一只手叉腰。
- 身体轻微前倾。
- 轮廓光测试站姿。

### v0.4b 追加拆分：人物与承载物联动

这个功能承接用户关于“人物可以放到桌子、椅子、小舞台上，也可以上下移动”的需求。当前 v0.3.4 只完成了高度放置，v0.4b 要让姿态也跟着合理变化。

优先级：

1. ✅（v0.4.4）椅子 / 沙发 / 凳子：选择“放到承载物”时，把人物移动到坐面中心，并应用坐姿预设。
2. ✅（v0.4.4）小舞台 / 台座 / 箱体 / 桌面：人物脚底落到台面高度，保持站姿或用户当前姿态。
3. ✅（v0.4.5）拖动 / 移动 / 旋转物体时，如果人物已绑定在该承载物上，人物跟随移动；删除承载物、回到地面或手动挪人时解绑。
4. 拖动人物靠近承载物时，后续可做轻量吸附；第一版仍可通过下拉选择，避免误吸附。
5. ✅（v0.4.4）坐姿第一版不做完整 IK，只需要大腿/小腿/躯干形成可读的“坐着”，用于判断灯光遮挡和影子。

> 当前实现说明：v0.4.5 的“放到承载物”会建立实时绑定。人物保存承载物 id、本地 X/Z 偏移和相对朝向；`setObjectPositionXZ` / `updateObject` / `rotateObject` 会重算绑定人物的世界位置和朝向；“回到地面”、删除承载物或手动改人物位置会解除绑定。换姿态但位置不变不会误解绑。

已新增字段：

```ts
supportObjectId?: string;
supportLocalOffset?: { x: number; z: number };
supportRotationOffset?: number;
```

## 6. v0.5 灯具器械预设库

### 目标

让用户不只选择“硬光/柔光/面光”，还能选择接近真实器械语义的灯具预设。

第一版不追求官方光度曲线精确复刻，先做导演可理解的器械语义。

详细实现规格见 `V0_5_FIXTURE_SPEC.md`。✅ v0.5.0 已通过用户真机验收（2026-06-20）：`applyFixturePreset` 已加在 `src/state/actions/lightActions.ts`，8 预设在 `src/data/fixturePresets.ts`，`LightPanel` 加器械下拉和能力标签。

当前灯光数量状态：

- 已支持多灯：默认 Key / Fill / Rim 三盏。
- 左侧灯光列表支持添加、复制、删除。
- 当前上限是 3 盏（`MAX_LIGHTS = 3`），满 3 后 UI 显示“满 3”。
- v0.5 灯具库第一版先不提高上限，避免“灯具语义”和“多灯管理”同时变复杂。
- 后期可以加更多光源，但放到开源之后单独做 v0.9：把上限提高到 6 或 8，并补灯光列表折叠、命名、分组和性能验收。

## 9. v0.8 多语言

### 目标

开源之后再加入多语言。多语言不抢 v0.5 / v0.6 / v0.7 主线，也不阻塞开源第一版。

首批语言：

- 简体中文。
- 英语。
- 日语。

### 为什么后置

- 当前阶段界面文案还会随着灯具、附件和开源前收口继续变化，过早做 i18n 会增加维护成本。
- 等开源第一版稳定后再统一抽文案，能避免反复翻译和反复改 key。
- 先保证简体中文版本能顺手使用，再做英语和日语。

### 实现原则

- 默认语言：简体中文。
- 语言切换放在设置或顶栏的轻量入口，不影响主操作。
- UI 文案、按钮、面板标题、空状态、导出说明都进入语言包。
- 灯具型号、品牌名、数值单位不翻译，只翻译类别和说明。
- 保存的方案数据不依赖语言，切换语言不改变场景内容。

### 写入范围建议

- 新增 `src/i18n` 或 `src/lib/i18n`。
- 新增 `zh-CN` / `en` / `ja` 文案表。
- UI 组件逐步改为读取文案 key。
- 文档保留中文主文档，后续可补英文 README 摘要。

### 暂时不做

- 自动机器翻译。
- 用户自定义语言包。
- 云端语言同步。
- 多语言内容编辑器。

### 验收标准

- 能在简体中文、英语、日语之间切换。
- 切换语言后，场景、A/B、保存方案不丢状态。
- 主要 UI 无明显溢出、遮挡或按钮文字挤压。
- 至少覆盖：顶部栏、对象列表、参数面板、A/B 引导、导出/保存相关文案。

## 12. v0.4.6 A/B 对比产品引导

> 状态：✅ 已通过 Codex 验收（Hermes 起草，Codex 复核修正，2026-06-19）。差异摘要、B 冻结时间、空状态三动作卡、A/B 文案更新都已接入。Codex 已修正一个流程问题：进入对比不再自动生成 B，因此空状态引导卡可以真实出现。

### 目标

让 A/B 从“一个可用视图”变成“导演沟通工具”：

- A 是正在编辑的当前方案，B 是冻结或加载的参考方案。
- 用户一眼知道 B 的来源：冻结当前、已存方案、交换前方案。
- 用户能快速开始一次对比，而不是先理解内部逻辑。
- 对比时能看到“这次主要在比什么”：灯位、人物位置、道具位置、姿态、摄影机或白棚。
- A/B 继续作为后续摄影机、灯具、控光附件的默认验收容器。

### MVP 范围

- 对比顶部状态更清楚：A 标为“当前编辑”，B 标为“冻结参考”。
- B 为空或刚进入对比时，给出明确的下一步动作：冻结当前为 B、选择已存方案、或回到普通视图继续调整。
- 冻结 B 后，显示 B 的时间/来源/名称，避免用户忘记 B 是什么。
- 交换 A/B 前后，文案要明确“交换后正在编辑哪一边”。
- 对比画面增加简短差异摘要：灯光、人物、道具、姿态、摄影机、白棚，先做大类提示，不追求完整 diff。
- 给 3 个导演常用对比入口：灯光前后、站位前后、道具/小舞台位置前后。

### 暂时不做

- 完整逐字段 diff 表。
- 多版本时间轴。
- 评论批注系统。
- 云端协作。
- 自动判断哪个方案更好。

### 设计原则

- 不写长教程；用清晰标签、按钮状态、空状态和短提示完成引导。
- A/B 的核心动作只保留少数几个：冻结、选择 B、交换、保存方案。
- 对比摘要服务于沟通，不服务于调试；只告诉导演“差异大概在哪类”。
- 所有引导文案都要短，不能遮挡画面。

### 验收标准

- 用户第一次进入对比视图时，不需要解释也能开始冻结 B。
- 冻结 B 以后，用户能说清楚“左边是我正在改的，右边是不动的参考”。
- 改灯位、人物位置或道具位置后，对比摘要能提示主要差异类型。
- 交换 A/B 后，用户不会误以为正在编辑 B。
